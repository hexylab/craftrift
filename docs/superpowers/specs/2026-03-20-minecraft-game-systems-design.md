# CraftRift ゲームシステム設計 — Minecraft要素融合

## 概要

CraftRiftのゲームシステムを、LoL ARAMの再現からMinecraft要素を中心に据えた独自システムに改訂する。ゴールド経済をMinecraft素材の採掘・クラフト経済に置き換え、チャンピオンスキルを武器切り替えシステムに簡素化し、ブロック建築を防御・地形操作の戦術手段として活用する。

### 設計原則

- **リソース競合**: 武器強化とブロック強化が同じ素材プールを奪い合い、戦略的選択を生む
- **ラストヒット報酬**: ミニオンへのラストヒットが経済の主要ドライバー
- **パラメータ駆動**: すべてのバランス値を定数として外出しし、チューニング容易に
- **段階的実装**: モジュール分割型で各システムを独立して実装・テスト可能

---

## 1. 素材＆ドロップシステム

### 1.1 素材定義

4種の素材: `Wood`（木）, `Stone`（石）, `Iron`（鉄）, `Diamond`（ダイヤ）

### 1.2 ドロップソース

| ソース | トリガー | ドロップ量 |
|--------|---------|-----------|
| ラストヒット | プレイヤーがミニオンにとどめ | 多い（例: 3〜5個） |
| 近接ドロップ | 味方ミニオンがキル＋プレイヤーが一定範囲内 | 少ない（例: 0〜1個） |
| プレイヤーキル | プレイヤーが敵プレイヤーを倒す | 最も多い（例: 5〜8個）※マルチプレイヤー実装後 |

### 1.3 時間経過によるドロップ確率変動

ゲーム開始時は木がほぼ100%。時間経過で上位素材の確率が緩やかに上昇する。

```
ゲーム開始時:  木=90%  石=8%   鉄=2%   ダイヤ=0%
5分後:         木=70%  石=20%  鉄=8%   ダイヤ=2%
10分後:        木=50%  石=28%  鉄=17%  ダイヤ=5%
15分後:        木=35%  石=30%  鉄=25%  ダイヤ=10%
20分後:        木=25%  石=28%  鉄=30%  ダイヤ=17%
```

確率の変動カーブはシグモイド的な滑らかな関数で実装する。具体的な数値はすべてパラメータとして外出し。

### 1.4 近接ドロップ判定

味方ミニオンが敵ミニオンをキルした時、プレイヤーが半径N（パラメータ、例: 12ブロック）以内にいれば少量の素材を獲得する。

### 1.5 ミニオンのグレードとドロップの連携

ミニオンのグレード（ウェーブ番号によるスケーリング）が上がると、ドロップ個数が増加する。

### 1.6 設計上のポイント

- `MaterialType` 列挙型（Wood, Stone, Iron, Diamond）を新設
- `DropTable` クラス: ゲーム経過時間を受け取り確率テーブルを返す
- ドロップ個数・確率・距離などすべてパラメータ定数として定義
- 敵プレイヤーキル＆アシスト判定は将来Issue（マルチプレイヤー時）

## 2. インベントリ＆クラフトシステム

### 2.1 インベントリ

- **素材スロット**: 4種（木・石・鉄・ダイヤ）それぞれのストック数。上限はパラメータ（例: 各99）
- **装備スロット**: 現在装備している武器5枠（素手・剣・斧・槍・弓）。各スロットに素材グレード（なし/木/石/鉄/ダイヤ）を保持
- **ブロックレベル**: 現在の設置可能ブロックのグレード（土/木/石/鉄/ダイヤ）

### 2.2 クラフト（武器強化）

累積コストによる段階的アップグレード。前段階の武器＋上位素材で強化する。

```
素手 → 木の剣 = 木×N
木の剣 → 石の剣 = 石×N
石の剣 → 鉄の剣 = 鉄×N
鉄の剣 → ダイヤの剣 = ダイヤ×N
```

4種の武器（剣・斧・槍・弓）すべて同じパターン。コストNは武器種ごとにパラメータで設定可能。

### 2.3 クラフト（ブロック強化）

武器と同じ素材プールからブロックグレードを上げる。

```
土（デフォルト） → 木ブロック = 木×M
木ブロック → 石ブロック = 石×M
石ブロック → 鉄ブロック = 鉄×M
鉄ブロック → ダイヤブロック = ダイヤ×M
```

コストMもパラメータ。武器とブロックが同じ素材を奪い合う設計。

### 2.4 クラフトゾーン

- ネクサス後方の本陣に**クラフトブロック**を床面に配置（新BlockType）
- プレイヤーがクラフトブロックの上に立っている時のみEキーでクラフトUI表示
- クラフトUI: 武器4種＋ブロック1種の強化メニュー
- 各項目: 現在グレード → 次グレード、必要素材、所持素材数、クラフトボタン
- 素材が足りない項目はグレーアウト

### 2.5 クラフトUI動作

- Eキーで開閉（クラフトゾーン上のみ）
- 開いている間はプレイヤー操作不能（ゲーム自体は進行し続ける。クラフトに時間をかけすぎるとミニオンを逃す）
- クラフトゾーンから離れると自動的にUIが閉じる

### 2.6 設計上のポイント

- `Inventory` クラス: 素材ストック・装備状態・ブロックレベルを一元管理
- `CraftingRecipe` 定義: 武器種×グレードごとのコストテーブル（パラメータ外出し）
- `CraftingSystem`: レシピ判定＋素材消費＋装備/ブロックアップグレード
- クラフトUIはHTML/CSSオーバーレイ（既存HUDパターンに準拠）
- クラフトゾーン判定は `MapData` にクラフトブロック配置範囲を定義

## 3. 武器システム

### 3.1 武器5種の攻撃判定

| 武器 | 判定方式 | 射程 | 威力 | 特殊 |
|------|---------|------|------|------|
| 素手 | 単体レイキャスト | 短（例: 3） | 最低 | なし |
| 剣 | 前方扇形（例: 90°） | 中（例: 4） | 中 | 範囲内の全敵にヒット |
| 斧 | 単体レイキャスト | 中（例: 4） | 高 | 単体特化 |
| 槍 | 直線レイ貫通 | 長（例: 6） | 中 | レイ上の全敵にヒット |
| 弓 | 放物線弾（低重力） | 最長（例: 30） | 低 | 弾速パラメータあり |

### 3.2 素材グレードによるステータス倍率（後半急峻カーブ）

| グレード | ダメージ倍率 | 攻撃速度倍率 |
|---------|-------------|-------------|
| 素手/なし | 1.0 | 1.0 |
| 木 | 1.3 | 1.0 |
| 石 | 1.8 | 1.05 |
| 鉄 | 2.8 | 1.15 |
| ダイヤ | 4.5 | 1.3 |

後半ほど急峻に上昇し、上位素材への集中投資に意味を持たせる。すべてパラメータ。

### 3.3 武器切り替え

- **数字キー**: 1=素手、2=剣、3=斧、4=槍、5=弓。固定位置。未クラフトの武器を選ぶと素手と同じ挙動
- **マウスホイール**: クラフト済み（＋素手）の武器間のみをスキップ切り替え
- HUDにホットバー表示（5枠固定、未クラフトはグレーアウト、選択中はハイライト）

### 3.4 ブロック破壊との関係

- 武器種・素材グレードによる破壊速度の差はなし
- 攻撃クールダウンと共有（ブロック殴り中は敵を攻撃できない）

### 3.5 設計上のポイント

- `WeaponType` 列挙型（Fist, Sword, Axe, Spear, Bow）
- `WeaponStats` 定義: 武器種×グレードの組み合わせでダメージ・射程・クールダウンを算出
- 攻撃判定は `CombatSystem` を拡張: 武器種に応じた判定方式を分岐
- 弓は既存 `Projectile`/`ProjectileManager` を拡張して重力付き矢を追加
- 倍率テーブルはパラメータ定数として外出し

## 4. ブロック強化＆設置システム

### 4.1 ブロックグレードと耐久度

| ブロック | 耐久値（殴る回数） |
|---------|-------------------|
| 土 | 1 |
| 木 | 3 |
| 石 | 6 |
| 鉄 | 10 |
| ダイヤ | 15 |
| 黒曜石 | 破壊不可 |

### 4.2 設置ルール

- 設置上限: パラメータ（例: 20個）。上限に達すると設置不可
- プレイヤーが設置したブロックを自分で壊すと設置枠が1つ回復
- 敵に壊された場合も枠が回復
- 設置できるのは現在のブロックグレードのブロックのみ（常に最新グレード）
- 素材は設置時に消費しない（アンロック時のみ消費）

### 4.3 ブロック耐久の管理

- 設置されたブロックごとに残り耐久値を記録
- 攻撃が当たるたびに耐久値-1
- 耐久値0で破壊
- ヒビ/ダメージ表現: Minecraft式にブロック表面にクラック段階を表示（3段階程度）

### 4.4 マップブロック構成

既存マップブロックも新耐久度システムに統合する。

| 場所 | ブロック | 破壊 |
|------|---------|------|
| 壁 | 黒曜石（新BlockType） | 不可 |
| 地表〜地下1ブロック（2層分） | 土・石・鉄・ダイヤをランダム配置 | 可（耐久度に従う） |
| 地下2ブロック目（3層目） | 黒曜石 | 不可 |
| 基盤（最下層） | 基盤 | 不可 |
| 構造物（タワー・ネクサス） | 既存のまま | 不可（CombatSystem経由で攻撃） |

### 4.5 地面ランダム配置の比率

ゲーム開始時にシード値でランダム生成。毎ゲーム異なるマップになる。

```
土=60%  石=25%  鉄=12%  ダイヤ=3%
```

比率はパラメータで調整可能。

## 5. ミニオン＆進行システム

### 5.1 ミニオンのステータススケーリング

| パラメータ | 初期値 | 上昇方式 |
|-----------|--------|---------|
| ミニオンHP | 150 | ウェーブ番号ごとに+N |
| ミニオン攻撃力 | 10 | ウェーブ番号ごとに+N |
| ウェーブサイズ | 3 | 一定ウェーブごとに+1（上限あり） |

スケーリング係数はすべてパラメータ。

### 5.2 ドロップ連携

ウェーブ番号が上がるとドロップ個数が増加（パラメータで調整）。

### 5.3 将来の拡張ポイント（今回は実装しない）

- Mob多様化（ゾンビ・スケルトン・クリーパー等）→ 将来Issue
- キャノンミニオン・スーパーミニオン → 将来Issue
- 現段階は羊モデルのまま、ステータスとドロップの仕組みだけ拡張

## 6. HUD＆操作統合

### 6.1 HUD新規要素

- **ホットバー**（画面下部中央）: 5枠固定（素手・剣・斧・槍・弓）。選択中ハイライト、未クラフトはグレーアウト、各武器のグレードアイコン表示
- **素材表示**（画面右下 or ホットバー上部）: 木・石・鉄・ダイヤの現在所持数
- **ブロック設置数**（ホットバー付近）: 「設置数 / 上限」表示
- **クラフト可能通知**: クラフトゾーン上にいる時「Eキーでクラフト」表示

### 6.2 HUD既存維持

プレイヤーHP、ターゲット情報、死亡画面、タワー警告、ダメージフラッシュ、勝利画面

### 6.3 クラフトUI

- Eキーで開閉（クラフトゾーン上のみ）
- 開いている間はプレイヤー操作不能（ゲームは進行し続ける）
- 武器4種＋ブロック1種の強化メニュー
- 各項目: 現在グレード → 次グレード、必要素材、所持素材、クラフトボタン
- 素材が足りない項目はグレーアウト

### 6.4 操作一覧

| キー | 動作 |
|------|------|
| WASD | 移動 |
| Space | ジャンプ |
| 左クリック | 攻撃 / ブロック破壊（装備武器に応じた判定） |
| 右クリック | ブロック設置 |
| 1〜5 | 武器直接選択（1=素手、2=剣、3=斧、4=槍、5=弓、固定位置） |
| マウスホイール | 武器切り替え（未所持スキップ） |
| E | クラフトUI開閉（クラフトゾーン上のみ） |
| F5 | 視点切替（FPS/TPS） |
| K | デバッグ: 50ダメージ自傷 |

## 7. パラメータ設計（バランス調整の要）

### 7.1 パラメータ集約

すべてのバランス定数を `src/config/GameBalance.ts` に集約。

```
DROP（ドロップ系）
├── ラストヒット報酬量（基本個数、ウェーブ番号ボーナス）
├── 近接ドロップ報酬量、判定半径
├── 素材確率テーブル（時間帯ごとの木/石/鉄/ダイヤ比率）
└── 確率変動カーブのパラメータ

WEAPON（武器系）
├── 武器種別ベースステータス（ダメージ、射程、クールダウン、扇形角度等）
├── グレード倍率テーブル（ダメージ倍率、攻撃速度倍率）
└── 弓パラメータ（弾速、重力係数）

CRAFT（クラフト系）
├── 武器種別クラフトコスト
├── ブロック強化コスト
└── 素材スタック上限

BLOCK（ブロック系）
├── グレード別耐久値
├── 設置上限
└── マップ地面ランダム配置比率

MINION（ミニオン系）
├── 基本HP/攻撃力/移動速度
├── ウェーブごとのスケーリング係数
└── ウェーブ間隔、サイズ、スポーン設定
```

### 7.2 設計方針

- **1ファイルに集約**: 散在する定数を `GameBalance.ts` にまとめ、チューニング時に1箇所を見ればよい状態にする
- **既存の定数を移行**: 現在各ファイルに散らばっている定数（MINION_HP、ATTACK_DAMAGE等）を GameBalance 経由に統一
- **型安全**: TypeScript の `as const` で定義し、タイポを防ぐ

## 8. 死亡時のペナルティ

- 死亡時にインベントリ（素材・装備）は失わない
- リスポーン時間のみがペナルティ（既存の5秒タイマー）

## 10. 実装順序（アプローチA: モジュール分割型）

```
1. GameBalance.ts パラメータ集約
   └── 既存定数の移行、新パラメータの定義
2. マップ改訂
   └── 黒曜石壁、ランダム地面、クラフトゾーン配置
3. ブロック耐久度システム
   └── 耐久値管理、クラック表示、マップブロック統合
4. 素材＆ドロップシステム
   └── MaterialType、DropTable、ラストヒット/近接ドロップ
5. インベントリ＆クラフトシステム
   └── Inventory、CraftingRecipe、CraftingSystem、クラフトUI
6. 武器システム
   └── WeaponType、攻撃判定分岐、武器切り替え、ホットバーHUD
7. HUD統合
   └── 素材表示、ブロック設置数、クラフト通知
```

各ステップはIssue単位でPRを切り、テスト・バランス調整が可能。
