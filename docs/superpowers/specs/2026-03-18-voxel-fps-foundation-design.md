# CraftRift: ボクセルFPS基盤 設計仕様書

## 概要

Minecraft風のボクセルワールドをWebブラウザ上で動作させる基盤を構築する。一人称視点でWASD移動、マウスで視点操作、ブロックの破壊・設置が可能。将来的にLeague of Legends（ARAM）のゲームシステムとマルチプレイヤー対戦を追加するための土台となる。

## スコープ

### 含まれるもの
- ボクセルワールドの描画（チャンクベース）
- WASD移動 + マウス視点操作（一人称視点）
- ブロックとの衝突判定（AABB）
- ブロックの破壊・設置
- ARAMレーン形状の固定マップ
- プレースホルダーテクスチャ（後から差し替え可能）

### 含まれないもの
- ジャンプ・重力
- マルチプレイヤー（WebSocket）
- ゲームシステム（タワー、ネクサス、HP、スキル）
- サウンド
- HUD（クロスヘア以外）

### 対象ブラウザ
- Chrome / Firefox 最新版（WebGL2対応必須）
- WebGL非対応時は画面にエラーメッセージを表示

## 技術スタック

- **言語:** TypeScript
- **3Dライブラリ:** Three.js
- **ビルドツール:** Vite
- **パッケージマネージャ:** npm

## ボクセルワールド設計

### チャンクシステム

- チャンクサイズ: 16×16×16ブロック
- データ構造: `Uint8Array(4096)` でブロックIDを保持
- インデックス計算: `x + z * 16 + y * 256`
- チャンクごとにメッシュを生成、ブロック変更時はそのチャンクのみ再構築

### メッシュ生成（Culled Meshing）

- 隣接ブロックに隠れている面は生成しない
- 隣接判定にはチャンク境界を跨ぐケースも含む。`World.getBlock(worldX, worldY, worldZ)` でグローバル座標から任意のブロックを取得し、チャンク境界の隣接判定に使用する
- 各露出面に対して2つの三角形（1つの四角形）を生成
- `BufferGeometry` で直接構築（位置、法線、UV、インデックス）
- Greedy Meshingは初期段階では不要

### ブロックタイプ

| ID | 名前 | 説明 | 破壊可能 |
|----|------|------|----------|
| 0 | AIR | 空気（描画なし） | - |
| 1 | GRASS | 草（上面/側面/底面でテクスチャが異なる） | Yes |
| 2 | DIRT | 土 | Yes |
| 3 | STONE | 石 | Yes |
| 4 | BEDROCK | 岩盤（マップ境界） | No |

### テクスチャシステム

- テクスチャアトラス方式: `TextureLoader.ts` がランタイムで `public/textures/` の個別PNG（16×16px）をCanvas APIを使い1枚のアトラステクスチャに合成する
- ブロックタイプごとに上面・側面・底面のUV座標をマッピング
- プレースホルダーは単色 + 簡易パターンのPNG
- `public/textures/` のファイルを差し替えるだけでテクスチャ変更可能

### 固定マップ（ARAMレーン）

- レーン: 中央にまっすぐ伸びる道（幅11ブロック、長さ80ブロック、Z軸方向に延伸）
- 拠点エリア: 両端にチーム拠点用スペース（将来のタワー・ネクサス配置用）
- 壁: レーン外側をBEDROCKの壁で囲む（高さ5ブロック）
- 地面構成: BEDROCK 1層 + DIRT 2層 + GRASS 1層（計4ブロック厚）
- 壁素材: STONE（内側面）、BEDROCK（外周）
- マップ全体サイズ: 15×10×84ブロック（壁含む）→ チャンク数: 1×1×6 = 6チャンク
- プレイヤー初期スポーン: レーン一方の拠点エリア中央、地面から2ブロック上

## プレイヤーとカメラ設計

### 一人称カメラ

- Pointer Lock API でマウスカーソルをキャプチャ（クリックで有効化、Escで解除）
- マウス移動で視点回転（yaw/pitch）、デフォルト感度: 0.002 rad/pixel（定数として変更可能）
- pitch制限: -89° ~ +89°
- カメラ位置 = プレイヤー位置 + 目の高さ（1.6ブロック）

### 移動

- W/S: カメラの向き方向に前進/後退（水平面のみ、Y軸方向の移動はなし）
- A/D: カメラに対して左右に平行移動
- 移動速度: 4.3 blocks/sec（Minecraftの歩行速度と同等）、加速・慣性なし
- 重力なし。プレイヤーのY座標はスポーン時の値で固定される
- AABB衝突判定: プレイヤーを幅0.6×高さ1.8×奥行0.6のバウンディングボックスとして扱い、ブロックとの貫通を防止する。移動時にワールドのブロックデータと照合し、ソリッドブロックへの侵入を許可しない

### ブロックインタラクション

- DDAアルゴリズム（Amanatides & Woo方式）でボクセルグリッドを走査してターゲットブロックを検出
- 最大距離: 5ブロック
- 左クリック: ターゲットブロックを破壊（AIRに変更）。BEDROCKは破壊不可
- 右クリック: ターゲットブロックの隣接面に新ブロックを設置（Phase 1ではSTONE固定。将来的にブロック選択機能を追加予定）
- ハイライト: ターゲットブロックにワイヤーフレーム枠を表示

### HUD

- 画面中央にクロスヘア（+マーク）のみ

## ゲームループとレンダリング

### ゲームループ

- `requestAnimationFrame` ベース
- `deltaTime` 計算でフレームレート非依存の移動速度
- 処理順序: 入力処理 → プレイヤー更新 → ワールド更新 → レンダリング

### レンダラー設定

- Three.js `WebGLRenderer`
- FOV: 75°
- 描画距離: 128ブロック（固定マップに十分）
- ウィンドウリサイズ時に自動調整

### パフォーマンス方針

- `BufferGeometry` で直接メッシュ構築
- テクスチャアトラスで1マテリアル → ドローコール最小化
- ブロック変更時は該当チャンクのみ再構築（隣接チャンク境界面も考慮）
- Three.jsのフラストラムカリングをそのまま活用
